## 未対応/差分パターン（57件）

## 1. 単独の大きめ算用数字を文中でひらがな化しない（34件）
- パターン: `100` 以上の裸数字（助数詞や通貨記号なし）
- 例:
  - 入力: `100` / 期待: `ひゃく` / 現行: `100`
  - 入力: `1234` / 期待: `せんにひゃくさんじゅうよん` / 現行: `1234`
  - 入力: `1000000` / 期待: `ひゃくまん` / 現行: `1000000`
- 補足: 現行 `replaceInText` は誤置換抑制のため、助数詞/接頭辞などのトリガーがない裸数字は置換対象外。

## 2. 曜日（固定語彙）を読みに変換しない（8件）
- パターン: `月曜日` などの曜日語
- 例:
  - 入力: `月曜日` / 期待: `げつようび` / 現行: `月曜日`
  - 入力: `月曜日の9時` / 期待: `げつようびのくじ` / 現行: `月曜日のくじ`
- 補足: 数値規則中心のため、曜日語彙辞書は未実装。

## 3. 序数助数詞 `位` が未対応（5件）
- パターン: `1位`, `2位`, `10位`, `1位を獲得`
- 例:
  - 入力: `1位` / 期待: `いちい` / 現行: `1位`
- 補足: `位` 用の助数詞ルールが現行 `counters.json` にない。

## 4. `第X章` の促音規則が不足（3件）
- パターン: `第1章`, `第8章`, `第10章`
- 例:
  - 入力: `第1章` / 期待: `だいいっしょう` / 現行: `だいいちしょう`
  - 入力: `第10章` / 期待: `だいじゅっしょう` / 現行: `だいじゅうしょう`
- 補足: `章` は現行 `concat` 扱いで、1/8/10 の促音化を持っていない。

## 5. `11歳` の読み方ポリシー差（1件）
- パターン: `11歳`
- 例:
  - 入力: `11歳` / 期待: `じゅういっさい` / 現行: `じゅういちさい`
- 補足: locotown 側は 11 を `いっ` 化、現行は通常連結。

## 6. `3箱` の読み方ポリシー差（1件）
- パターン: `3箱`
- 例:
  - 入力: `3箱` / 期待: `さんばこ` / 現行: `さんぱこ`
- 補足: 現行は要件に合わせて `さんぱこ` を採用済み。

## 7. 高い数値の助数詞変換ポリシー差（1件）
- パターン: `100歳`
- 例:
  - 入力: `100歳` / 期待: `100歳` / 現行: `ひゃくさい`
- 補足: locotown 側は 100以上を未変換、現行は変換対象。

## 8. 円の扱い（単位を残すか読み切るか）のポリシー差（2件）
- パターン: 円を含む文の変換仕様
- 例:
  - 入力: `価格は300円です` / 期待: `価格はさんびゃく円です` / 現行: `価格はさんびゃくえんです`
  - 入力: `100円と200円` / 期待: `ひゃく円とにひゃく円` / 現行: `ひゃくえんとにひゃくえん`
- 補足: locotown 側は数字だけ読み化、現行は助数詞（円）まで読み化。

## 9. 非助数詞単位付き数値の変換（1件）
- パターン: `1000メートル`
- 例:
  - 入力: `距離は1000メートル` / 期待: `距離はせんメートル` / 現行: `距離は1000メートル`
- 補足: 現行は `メートル` を助数詞定義していないため未変換。
